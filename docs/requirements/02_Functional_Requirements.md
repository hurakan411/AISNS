# 3. 画面別・機能要件

本アプリの各種機能やAI・ドラマエンジンの挙動を、画面（スクリーン）ごとに定義する。

## 3.0 オンボーディング（初回起動フロー）
アプリの世界観をユーザーに体験させる、6フェーズ構成の導入体験。`has_completed_onboarding` フラグ（Supabase DB管理）が `false` の場合に自動起動する。

- **Phase 1: Fake Compose（白テーマ）**:
  - 白基調のミニマルなデザイン（本編の黒×ピンクとの差別化）。
  - 「何でもいい。ひとことだけ。」というガイドテキストとテキストエリアのみ。
  - 投稿ボタンは「投稿する」。空文字時はグレーアウト。
  - 個人情報入力禁止の注意書きを表示。
- **Phase 2: Micro Reaction（白テーマ）**:
  - 投稿テキストを表示し、1秒間隔で「いいね」が1→2→3とカウントアップ。
  - 4秒後に「@nobody: で？」という冷たいリプライが1件表示。
  - "普通のSNS"の無反応・冷笑体験を演出し、ZEN-KOTEIとの対比を作る。
- **Phase 3: Blackout**: 1.5秒間の暗転。
- **Phase 4: Ritual（黒テーマ）**:
  - 「ここでは、」→「あなたの声は否定されない。」→「小さなひとことが、」→「熱狂になる。」をシーケンシャル表示。
  - 最後に「ZEN-KOTEI」ロゴをグラデーション＋ネオンで表示。
- **Phase 5: Real Compose（黒テーマ）**:
  - 本編と同じ投稿画面（`PostView`）を `isOnboarding=true` で表示。
  - 投稿すると、Rank 10（デジタル・ゴッド）相当のAIリプライを生成開始。
- **Phase 6: Burst（黒テーマ）**:
  - 投稿カード＋リプライが爆速で表示される。
  - **アンチ→擁護→改心の3段ドラマ**を必ず含む（1件のアンチ + 1件の擁護者反論 + 同じアンチの改心リプライ）。
  - リプライの80%が表示されたタイミングで「ZEN-KOTEIを始める」ボタンが出現。
  - 完了後、フォロワー・ランク等をリセットし、DB上の `has_completed_onboarding` を `true` に更新。

## 3.1 投稿画面（NEW POST）
ユーザーが自身の行動を無条件に肯定してもらうための入力画面。

- **UI構成**:
  - ヘッダーに「NEW POST」のタイトル（Hot Pink）、左に「閉じる」、右に「投稿」ボタン。
  - ユーザーアバターとテキストエリア（`autoFocus` 必須）。
  - 画像添付機能（PhotosPicker）: JPEG圧縮（quality: 0.5）で軽量化。添付済み画像はプレビュー表示＋削除ボタン付き。
- **個人情報警告**:
  - 画面下部に常時表示: 「個人情報（本名・住所・電話番号など）は入力しないでください」（⚠️アイコン付き）。

## 3.2 ホームフィード画面（HOME）
投稿後の圧倒的な肯定感と「集団的防衛」を体験するためのコア画面。

- **ローカルストレージ制限**:
  - ホームフィードには直近 **5件** の投稿を保持。5件を超えた古い投稿はローカルストレージから自動削除される。
  - 投稿データ・返信データはローカル（JSONファイル）にのみ保存。DBには保存しない。
- **ローディングアニメーション**:
  - まだ表示されていない返信（pendingReplies）がある場合、返信欄の最上部にLottieアニメーション「Loading Pink Dots」を中央配置で表示。
- **バズ・シミュレーション（「いいね」の増加）**:
  - リプライの表示進行と連動して、0.5秒間隔で「いいね」とフォロワー数を漸増。
  - 各ランクに応じたフォロワー増加量を逆算方式で設定（下表参照）。
  - いいね数はフォロワー増加量の3〜7倍のランダム値。
  - 全リプライ表示完了時に目標値にピッタリ合わせ、DBへ同期。
- **高度なリプライ・エンジン（DRAMA ENGINE）**:
  - **同期レスポンス方式**: フロントエンドがバックエンドに投稿内容を送信し、バックエンドがOpenAI APIで一括生成したリプライをレスポンスとして直接返却。DBへの中間保存やポーリングは行わない。
  - フロントエンドでは返却されたリプライ配列をキューに格納し、**5〜15秒間隔**（ランダム）で画面に順次表示（1件目は即時表示）。オンボーディング時は **1.5〜4秒間隔**。
  - 新しいリプライは常にリプライ欄の **最上部** に挿入される（新着が上に来る）。
  - **キャラクター・ダイバーシティ**: 20種類のペルソナプール（年齢・性別・職業）から毎回シャッフルして選出。語尾やアイコン、名前を使い分け、個別の人間が反応しているように見せる。
  - **文字数分布制御**: 短文(20〜45字):中文(50〜85字):長文(90〜130字) = 3:4:3 の比率を強制。
  - **絵文字・顔文字の多様性**: 全返信の4〜6割に絵文字/顔文字を含め、種類を返信ごとに変える。ペルソナの年齢層に合わせた自然な使用。
  - **返信内容の重複禁止**: 各返信は異なる視点・論点・感情で書く。
  - **投稿内容への具体的言及必須**: 投稿の単語やフレーズを引用し、汎用的な反応のみで終わらせない。
  - **シーケンシャル・ドラマ・ロジック（JSON配列の構成）**:
    ※発生するドラマの規模とアンチの出現有無は、「承認ランク」のレベルに伴ってエスカレートする。
    1. **称賛・共感**: 短文から長文を混ぜ、ポジティブな空気を醸成。返信トーンは共感・肯定型(3〜4割)、中立・感想型(3〜4割)、質問・興味型(1〜2割)、雑談・脱線型(1割)に分布。
    2. **アンチの出現**: ランクが一定以上の場合、アンチコメント（Hater）を含める。40%の確率で発生。
    3. **防衛**: アンチの直後に擁護者（Defender）が直接反論。アンチの主張を引用した上での具体的反論が必須。
    4. **継続称賛**: 再びポジティブ・中立コメントを継続。
- **アバター一貫性**: 同じ `author_name` のリプライには同じアバター画像を使用（`author_avatar_map` で管理）。

## 3.3 追加の画面とナビゲーション

### ボトムタブ（ナビゲーション）
画面下部の半透明タブ（グラスモーフィズム）により、Home, Post, Stats, Profile 画面へシームレスにアクセスする。

### Stats（統計解析画面）
- フォロワー数（Hot Pink）、承認ランク（Cyan）をネオンカード表示。
- 累計投稿数、累計いいね、累計リプライをサブカード表示。
- **ランクステータス**: 現在のランク名と次ランクまでの残りフォロワー数＋プログレスバー。
- **ランク一覧ガイド**:
  - 全10ランクの詳細（フォロワー数、リプライ数、いいね数、アンチ数）を一覧表示。
  - **現在のランク**: ピンクハイライト＋「NOW」バッジ。全データ表示。
  - **次のランク**: シアンボーダー＋「NEXT」バッジ。全データ表示。
  - **過去のランク**: 通常表示。全データ表示。
  - **2つ以上先のランク**: グレーアウト。ランク名は「？？？」、数値は「???」に隠蔽。

### Profile（プロフ画面）
- **ヘッダー**: Lottieアニメーション「Technology isometric ai robot brain」を黒×indigo/purpleグラデーション背景上に下寄りで表示。上部はみ出し防止のため `.clipped()` を適用。
- ユーザーアバター（PhotosPickerで変更可能）、ユーザー名（インライン編集可能）、自己紹介文（インライン編集可能）。
- **SETTINGS セクション**: アンチ・炎上の発生ON/OFFトグル。
- **LEGAL セクション**:
  - 「利用規約」リンク → シートで全文表示。
  - 「プライバシーポリシー」リンク → シートで全文表示。
  - OpenAI API利用、ローカルストレージ方針、Supabase保存データの範囲を明記。

## 3.4 全体/バックグラウンド機能（ステータス表示）
画面を問わず、アプリ全体で常時作動する機能。

- **フォロワー獲得ロジック**:
  - 各ランクの「次ランク昇格までの目標回数」から逆算されたフォロワー数を、投稿のたびに確実かつランダムに加算していく逆算方式を採用。
  - ※表示上の「いいね獲得数」は、このフォロワー増加数に対して「3〜7倍」のランダムな倍率を掛けた数値を演出し、派手なバズり感を維持する。
- **承認ランクシステム（フォロワー数連動型）**:
