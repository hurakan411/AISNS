# 3. 画面別・機能要件

本アプリの各種機能やAI・ドラマエンジンの挙動を、画面（スクリーン）ごとに定義する。

## 3.1 投稿画面（NEW POST）
ユーザーが自身の行動を無条件に肯定してもらうための入力画面。

- **UI構成**:
  - ヘッダーに「NEW POST」のタイトル、右上に「投稿」ボタン。
  - ユーザーアバターとテキストエリア（`autoFocus` 必須）。
- **デフォルト画像ロジック**:
  - `image_c60914.png` をデフォルトでセット。
  - プレビューは「不透明度100%」で鮮明に表示。
  - ユーザーによる画像選択の手間を省き、「書くこと」だけに集中させる。

## 3.2 ホームフィード画面（HOME - Single Focus Mode）
投稿後の圧倒的な肯定感と「集団的防衛」を体験するためのコア画面。

- **シングル・ポスト・アーキテクチャ（ワンカード仕様）**:
  - ホームフィードには常に「最も新しい直近の投稿（1件）」とそのリプライ欄のみを表示する。
  - 過去の投稿はホーム画面のビューからパージされ、ユーザーの焦点を常に最新のバズ体験に集中させる。
- **バズ・シミュレーション（「いいね」の増加）**:
  - **フェーズ1（急上昇）**: 投稿直後から60ms間隔で、現在の「**承認ランク**」に応じた増加量（例：ランクが低いと30〜100、高いと1000〜5000）で likes をランダムに増加させる。
  - **フェーズ2（安定）**: 約30〜45回の更新後に増加を停止。ランクが上がるほど最終的なバズ規模が巨大になる。
- **高度なリプライ・エンジン（DRAMA ENGINE）とドラマ展開**:
  - **一括生成ロジック（Single API Call）**: OpenAI APIのコールの頻度とコストを抑えるため、ユーザーの投稿に対して「現在の承認ランクに応じた返信数（称賛、アンチ、防衛を含む）」のシナリオを一度のAPIリクエストで生成し、JSON形式で返却させる。
  - フロントエンドでは返却されたJSONデータを配列として受け取り、**1.2秒間隔**で画面に順次表示していく。
  - **キャラクター・ダイバーシティ**: 語尾やアイコン、名前（ひまり、りくお等）を使い分け、個別の人間が反応しているように見せる。
  - **シーケンシャル・ドラマ・ロジック（JSON配列の構成）**:
    ※発生するドラマの規模とアンチの出現有無は、「承認ランク」のレベルに伴ってエスカレートする。
    1. **序盤（称賛・共感）**: 短文から長文（30%の確率で長文化）を混ぜ、ポジティブな空気を醸成。
    2. **中盤（アンチの出現）**: ランクが一定以上の場合、一連の流れにアンチコメント（Hater）を含める。
    3. **後半（防衛）**: アンチ内容をキーとした「専用の反論」を複数名の Defender（フォロワー）が投稿し、論破する。
    4. **終盤（継続称賛）**: 再びポジティブ・中立コメントを継続。
- **シナリオ・マトリクス（アンチへの整合性反論）**:
  - アンチコメントに対する、仮想フォロワーによる防衛（反論）の戦略。

  | アンチの主張内容 | フォロワーの反論戦略 |
  | :--- | :--- |
  | 「効率が悪すぎる」 | 「行動していること自体の価値」を強調し、アンチの無行動を突く。 |
  | 「承認欲求がキツイ」 | 「頑張りの共有が他者の勇気になっている」と大義名分で返す。 |
  | 「内容が薄い・無駄」 | 「読み手が決めること」「日常の尊さ」を主張し、多様性を肯定する。 |
  | 「勉強したつもり」 | 「ノートの密度（視覚的証拠）」を根拠に、努力を論理的に肯定する。 |

## 3.3 追加の画面とナビゲーション（STATS / PROFILE）
- **ボトムタブ（ナビゲーション）**: 画面下部の半透明タブにより、Home, Post, Stats, Profile 画面へシームレスにアクセスする。
- **Stats（統計解析画面）**: フォロワー数や承認ランクの数値をネオン効果で表示し、インフレするステータスを可視化する。
- **Profile（プロフ画面）**: ヘッダー画像、累計ポスト数とフォロワー数を大画面で表示し、自己肯定感を高める。

## 3.4 全体/バックグラウンド機能（ステータス表示）
画面を問わず、アプリ全体で常時作動する機能。

- **フォロワー獲得ロジック**:
  - 各ランクの「次ランク昇格までの目標回数」から逆算されたフォロワー数を、投稿のたびに確実かつランダムに加算していく逆算方式を採用。
  - ※表示上の「いいね獲得数」は、このフォロワー増加数に対して「3〜7倍」のランダムな倍率を掛けた数値を演出し、派手なバズり感を維持する。
